Configuration Sync
Automatically sync FGUARD UTC config from MASTER to BACKUP
Requirement: SSH key-based auth must be set up: ssh-copy-id root@<BACKUP_IP>
Setup Guide
MASTER Node
  1. Set Role = MASTER, Priority = 100
  2. Set Virtual IP (e.g. 10.0.0.254/24)
  3. Set VRRP Router ID (e.g. 51)
  4. Set Peer IP = BACKUP node's LAN IP
  5. Set Auth Password (same on both)
  6. Enable Auto Sync, set Sync Target = BACKUP IP
  7. Run: ssh-copy-id root@<BACKUP_IP>
  8. Click Apply / Start
BACKUP Node
  1. Set Role = BACKUP, Priority = 90
  2. Same Virtual IP as MASTER
  3. Same VRRP Router ID as MASTER
  4. Set Peer IP = MASTER node's LAN IP
  5. Same Auth Password as MASTER
  6. Click Apply / Start
Both nodes must have apt install keepalived. Clients connect to the Virtual IP — it automatically moves to the active MASTER.
