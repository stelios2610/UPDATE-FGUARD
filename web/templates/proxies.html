Proxy Rules intercept and inspect traffic at the application layer before it reaches the destination. HTTP/HTTPS proxies can filter web content; SMTP proxies scan email for malware and spam; DNS proxies can block domains at the resolver level; FTP proxies inspect file transfers. Rules are applied in order — the first matching rule wins.

HTTP / HTTPS Proxy
Intercepts web traffic on ports 80/443. Supports URL pattern matching, content inspection, SSL inspection (HTTPS), and category-based filtering. Can block, allow, or redirect requests based on URL, domain, MIME type, or content.
SMTP Proxy
Inspects email traffic on port 25/587/465. Blocks spam, scans attachments for malware, enforces sender/recipient policies, and can quarantine suspicious messages. Supports SPF/DKIM validation.
DNS / FTP Proxy
DNS proxy intercepts DNS queries on port 53 for domain-level blocking and logging. FTP proxy inspects file transfer protocol traffic on port 21, scanning uploaded/downloaded files for malware and enforcing transfer policies.
